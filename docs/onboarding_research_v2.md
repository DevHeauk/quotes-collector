## 1. 핵심 발견

### 한국 시장에서의 기회

**감정/니즈 기반 분류를 적용한 한국 명언 앱은 0개입니다.**
- GoodWords: 동기부여, 자기계발, 인생, 성공, 노력, 사랑, 위로, 힐링, 지혜, 속담...
- 오늘의 명언: 사랑, 공부 등 테마별
- → 전부 **주제 기반 추상 카테고리**. 차별화 없음.

반면 글로벌 앱들은 이미 전환 완료:
- **Olimp**: "무엇이 당신을 가로막고 있나요?" (두려움 기반)
- **Headspace**: "왜 왔어요?" (목적 기반)  
- **Monkey Taps Motivation**: 명언 톤 선택 (따뜻한 위로 vs 현실 직시)

**→ 한국 시장에서 "니즈 기반 분류"를 처음 적용하면 그 자체가 차별화**

### 우리 데이터가 말하는 것

콘텐츠 페르소나 분포 (1,245개 명언 분석):

| 페르소나 | 비율 | 대표 키워드 클러스터 |
|---|---|---|
| 학습형 | 29% | 학습+지식+지혜+겸손+교육 |
| 동기부여형 | 29% | 끈기+노력+도전+용기+행동 |
| 성찰형 | 26% | 철학+존재+자아+의미+자기성찰 |
| 관계형 | 19% | 사랑+관계+우정+가족+헌신 |
| 위로형 | 15% | 고통+회복+희망+외로움+두려움 |
| 유머형 | 6% | 유머+위트 |

→ 6개 자연 클러스터가 존재. 기존 8개 상황 그룹보다 이 클러스터가 사용자 세그먼트와 더 잘 맞음.

## 2. 제안: 3축 개인화 모델

기존의 `상황 그룹 + 키워드 그룹` 2축을 버리고, **니즈(Need) + 톤(Tone) + 시간(Time)** 3축으로 재설계합니다.

### 축 1: 니즈 (Need) — "왜 명언이 필요한가?"

사용자의 현재 상태/목표. 온보딩에서 수집.

| 니즈 | 사용자 언어 | 내부 매핑 |
|------|-----------|----------|
| 🔥 힘을 내고 싶어요 | "의지가 약해질 때 불을 지펴주는 말" | 동기부여형 (끈기, 노력, 도전, 용기, 행동) |
| 🌊 위로가 필요해요 | "힘든 시기를 견딜 수 있게 해주는 말" | 위로형 (고통, 회복, 희망, 외로움) |
| 🪞 나를 돌아보고 싶어요 | "삶의 의미를 생각하게 해주는 말" | 성찰형 (철학, 존재, 자아, 의미, 자기성찰) |
| 💡 시야를 넓히고 싶어요 | "새로운 관점을 열어주는 말" | 학습형 (학습, 지식, 지혜, 겸손) |
| 💛 사람 사이에서 | "관계에 대해 생각하게 해주는 말" | 관계형 (사랑, 관계, 우정, 가족) |
| 😄 웃고 싶어요 | "위트 있는 한마디" | 유머형 (유머, 위트) |

**6개. 데이터 자연 클러스터와 1:1 대응.**

### 축 2: 톤 (Tone) — "어떤 스타일을 좋아하는가?"

같은 "힘을 내고 싶어요" 니즈라도, 사용자마다 원하는 톤이 다릅니다:
- 따뜻한 위로형: "괜찮아, 천천히 가도 돼"
- 현실 직시형: "그만 울고 일어나"
- 철학적 성찰형: "고통은 성장의 전제조건이다"
- 짧고 강렬한: "Just do it"

**→ 온보딩이 아니라, 행동 데이터로 학습** (like/unlike 패턴에서 톤 선호 추출).

| 톤 | 특징 | 판별 기준 |
|---|---|---|
| warm | 공감, 위로, 부드러운 | 명언 길이 중간+ |
| tough | 직설, 도전, 각성 | 명령형, 짧은 문장 |
| deep | 철학적, 성찰적 | 추상 개념, 긴 문장 |
| witty | 유머, 반전, 위트 | 모순/역설 구조 |

### 축 3: 시간 (Time) — "언제 보는가?"

같은 사용자도 시간대에 따라 필요한 명언이 다릅니다.

| 시간 | 적합한 톤/니즈 |
|------|-------------|
| 아침 (6-10시) | 동기부여, tough/warm |
| 점심 (11-14시) | 학습, witty |
| 저녁 (18-22시) | 성찰, 위로, warm/deep |
| 심야 (22시+) | 위로, deep |

**→ 자동 적용. 사용자 선택 불필요.**

### 제안 B: 심층 (Olimp 스타일)

```
[Screen 1] 니즈 선택 (6개 중 최대 3개)
    ↓
[Screen 2] 장벽 선택 (선택한 니즈에 맞는 세부 장벽)
```

## 4. 데이터 구조 변경안

### 현재 (AS-IS)

```
quotes.keywords → JSONB ["용기", "도전"]        → keywords 마스터 (56개, 8그룹)
quotes.situation → JSONB ["포기하고 싶을 때"]     → situations 마스터 (40개, 8그룹)

온보딩 저장: situation_groups[], keyword_groups[]
```

### 제안 (TO-BE)

```
quotes.keywords → 유지 (세부 태깅용)
quotes.situation → 유지 (세부 태깅용)
quotes.need_type → NEW: 니즈 유형 (6종)         ← 기존 태그에서 자동 도출 가능
quotes.tone → NEW: 톤 (4종, optional)            ← 향후 태깅

온보딩 저장: needs[] (니즈 6종에서 선택)
```

### need_type 자동 매핑 규칙

기존 키워드/상황에서 need_type을 역산:

```python
NEED_MAPPING = {
    "motivation": {
        "keywords": ["끈기", "노력", "도전", "용기", "행동", "인내", "열정",
                     "목표", "성공", "실패", "실천", "동기부여"],
        "situations": ["게으를 때", "꾸준함이 필요할 때", "목표가 멀게 느껴질 때",
                       "포기하고 싶을 때", "도전을 망설일 때"]
    },
    "comfort": {
        "keywords": ["고통", "회복", "희망", "외로움", "두려움"],
        "situations": ["힘든 시기를 보낼 때", "절망적일 때", "좌절했을 때",
                       "실패했을 때", "외로울 때", "희망이 필요할 때",
                       "힘든 상황에서 거리를 두고 싶을 때", "불운할 때", "두려울 때"]
    },
    "reflection": {
        "keywords": ["철학", "존재", "자아", "의미", "자기성찰",
                     "인생", "시간", "죽음", "운명", "선택", "초월"],
        "situations": ["자기 성찰", "삶의 의미를 찾을 때", "인생의 선택",
                       "과거를 돌아볼 때", "죽음을 생각할 때", "현재를 살고 싶을 때",
                       "변화를 마주할 때", "미래가 불안할 때"]
    },
    "insight": {
        "keywords": ["학습", "지식", "지혜", "겸손", "교육", "창의성"],
        "situations": ["배움의 자세", "지식의 가치", "깊이 이해하고 싶을 때",
                       "공부하기 싫을 때", "새로운 관점이 필요할 때",
                       "과학적 사고", "창의적 사고"]
    },
    "relationship": {
        "keywords": ["사랑", "관계", "우정", "가족", "헌신", "감사", "공동체"],
        "situations": ["관계의 소중함", "관계가 어려울 때",
                       "사랑을 느낄 때", "사랑의 본질을 고민할 때", "감사할 때"]
    },
    "humor": {
        "keywords": ["유머", "위트"],
        "situations": ["웃음이 필요할 때", "일상의 소소함", "기분 전환이 필요할 때"]

```

### 마이그레이션 계획

1. `quotes` 테이블에 `need_types VARCHAR(20)[]` 컬럼 추가
2. 기존 keywords/situation 기반으로 need_types 자동 채움 (위 매핑)
3. 온보딩 코드에서 need_types 기준 추천으로 전환
4. **기존 keywords/situation은 그대로 유지** (세부 추천에 활용)

## 5. 추천 엔진 변경

### 현재
```
recommend(situations=["위기/극복"], keywords=["감정"])
→ situations 그룹 매칭 보너스 + keywords 그룹 매칭 보너스
```

### 변경 후
```
recommend(needs=["motivation", "comfort"])
→ need_types 매칭 보너스 (1차)
→ 기존 situation/keyword 매칭 (2차, 다양성 확보)
→ 시간대별 톤 가중치 (3차)
```

**1차 need_types 매칭이 가장 강한 시그널**, 2차/3차로 다양성과 시의성을 확보.
