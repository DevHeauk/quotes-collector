## 1. 현재 온보딩 진단

### 현재 플로우 (3단계)
1. **상황 선택** (6개 중 최대 3개): 위기/고난, 성장/도전, 관계, 자기성찰, 일/학업, 일상
2. **주제 선택** (9개 중 최대 3개): 감정, 가치관, 행동/태도, 관계, 지성, 인생, 사회, 자연/과학, 유머/표현
3. **명언 맛보기** (추천 명언 3개 좋아요)

### 문제점

| 문제 | 심각도 | 근거 |
|------|--------|------|
| Stage 2(주제) 선택이 추천에 기여하지 않음 | 🔴 높음 | "감정", "가치관", "인생" 같은 추상 카테고리는 대부분의 명언에 해당 → 차별화 불가 |
| Stage 1(상황)이 seed_groups.sql 기준이라 DB 마스터와 불일치 | 🔴 높음 | 현재 DB 상황 그룹은 8개(위기/극복, 시작/도전, 동기부여, 사랑/관계, 인생/성찰, 학습, 과학/탐구, 유머/일상)인데 온보딩은 다른 6개를 사용 |
| 2단계를 거쳐도 수집하는 시그널이 빈약 | 🟡 중간 | 6개 중 3개 + 9개 중 3개 = 6개 시그널인데, 사실상 상황 3개만 추천에 영향 |
| "관계"가 Stage 1과 Stage 2에 중복 등장 | 🟡 중간 | 사용자 혼란 유발 |

## 2. 경쟁 앱 벤치마킹

### 핵심 벤치마크 3개

#### Headspace (명상) — "극도의 간결함"
- **3단계, 1분 이내 완료**
- 경험 수준 → 사용 이유(Focus/Stress/Sleep/Anxiety) → 언제 할 건지
- 핵심: **"왜 왔어요?"** 하나로 충분

#### Olimp (명언 앱) — "심층 개인화"
- **다단계 퀴즈** (목표 + 두려움 + 멘토 선호)
- 28가지 두려움 목록 (실패 두려움, 미루기, 자기 의심, 불안 등)
- 핵심: **"두려움"을 물어서 감정적 연결** → 매우 높은 전환율

#### Blinkist (책 요약) — "즉시 보상"
- 관심 주제 선택 → **바로 맞춤 콘텐츠 보여줌**
- 핵심: **선택 직후 "당신을 위한 콘텐츠"** 즉시 표시

### 공통 패턴

| 패턴 | 사용 앱 | 효과 |
|------|---------|------|
| "왜 왔어요?" (목표 기반) | Headspace, Calm, Fabulous | 가장 적은 질문으로 최대 시그널 |
| "지금 어떤 상태?" (상황 기반) | Stoic, Olimp | 감정적 연결 강함 |
| 선택 직후 콘텐츠 표시 | Blinkist, Spotify | 빈 화면 방지, 즉각적 가치 체험 |
| 체험 먼저, 질문 나중 | Calm (심호흡), Stoic (첫 리플렉션) | 앱 가치를 먼저 느끼게 함 |

### 실패 패턴 (우리가 하고 있는 것)

| 패턴 | 문제 |
|------|------|
| 추상적 카테고리 나열 | "인생", "감정" 같은 것은 누가 골라도 비슷 |
| 카테고리 기반 2단계 | 사용자 피로만 늘리고 시그널 빈약 |
| DB 그룹명을 그대로 노출 | 내부 분류 체계는 사용자 언어가 아님 |

## 3. 제안: 온보딩 리디자인

### 핵심 원칙

1. **1단계로 통합** — Stage 2(주제)를 없앤다
2. **상황을 사용자 언어로** — DB 그룹명이 아니라 공감 가능한 문장으로
3. **Stage 3(맛보기)를 더 강화** — 선택 직후 맞춤 명언을 보여주는 것이 핵심 가치

### 안 A: "지금 필요한 말" (추천)

Headspace + Olimp 하이브리드. **1단계 상황 선택 → 즉시 맛보기**.

```
지금 당신에게 필요한 말은? (최대 3개)

😮‍💨  힘든 시기를 보내고 있어요         → 위기/극복
💪  포기하고 싶은 순간이에요           → 위기/극복 + 동기부여
🔥  새로운 도전을 앞두고 있어요        → 시작/도전
💔  관계에서 고민이 있어요             → 사랑/관계
🪞  나를 돌아보고 싶어요              → 인생/성찰
😂  웃음이 필요해요                   → 유머/일상
📚  공부/일에 동기가 필요해요          → 학습 + 동기부여
🌅  삶의 의미를 찾고 싶어요           → 인생/성찰
```

**DB 매핑 로직:**

| 온보딩 선택지 | situation 그룹 매핑 | 보조 keyword 그룹 매핑 |
|---|---|---|
| 힘든 시기를 보내고 있어요 | 위기/극복 | 고통, 회복, 희망 |
| 포기하고 싶은 순간이에요 | 위기/극복 + 동기부여 | 끈기, 인내, 용기 |
| 새로운 도전을 앞두고 있어요 | 시작/도전 | 도전, 용기, 변화 |
| 관계에서 고민이 있어요 | 사랑/관계 | 사랑, 관계, 우정 |
| 나를 돌아보고 싶어요 | 인생/성찰 | 자기성찰, 자아, 의미 |
| 웃음이 필요해요 | 유머/일상 | 유머, 위트 |
| 공부/일에 동기가 필요해요 | 학습 + 동기부여 | 학습, 노력, 목표 |
| 삶의 의미를 찾고 싶어요 | 인생/성찰 | 존재, 의미, 철학 |

**장점:**
- 1단계로 끝 → 이탈률 감소
- 사용자 언어로 공감 → 감정적 연결
- DB 상황 8그룹을 모두 커버
- 기존 추천 엔진(`_build_personalization`)과 호환

**콘텐츠 커버리지 검증:**

| 온보딩 선택지 | 매핑되는 명언 수 (중복 포함) |
|---|---|
| 힘든 시기를 보내고 있어요 | ~440건 (위기/극복 전체) |
| 포기하고 싶은 순간이에요 | ~200건 (위기/극복 + 동기부여) |
| 새로운 도전을 앞두고 있어요 | ~290건 (시작/도전) |
| 관계에서 고민이 있어요 | ~270건 (사랑/관계) |
| 나를 돌아보고 싶어요 | ~600건 (인생/성찰) |
| 웃음이 필요해요 | ~130건 (유머/일상) |
| 공부/일에 동기가 필요해요 | ~370건 (학습 + 동기부여) |
| 삶의 의미를 찾고 싶어요 | ~350건 (인생/성찰 일부) |

→ 어떤 조합이든 충분한 콘텐츠 확보

### 안 B: "목표 기반" (대안)

Fabulous 스타일. 상황 대신 **사용 목표**를 물음.

```
매일명언으로 무엇을 하고 싶으세요?

🧘  마음의 평화를 찾고 싶어요
💪  동기부여를 받고 싶어요
🧠  새로운 관점을 얻고 싶어요
❤️  사랑과 관계에 대해 생각하고 싶어요
😄  매일 웃고 싶어요
📖  지혜를 쌓고 싶어요
```

**장점:** 더 포지티브한 톤
**단점:** 감정적 연결이 약함, "동기부여"를 안 원하는 사람이 거의 없어서 차별화 약함

### 추천: 안 A

이유:
1. **"힘든 시기"를 고른 사람과 "웃음이 필요"를 고른 사람은 완전히 다른 명언을 받아야 함** → 상황 기반이 추천 차별화에 훨씬 강력
2. Olimp/Stoic 등 명언 앱에서 실제로 성과가 검증된 패턴
3. 우리 추천 엔진이 이미 상황 가중치 기반 → 구현 변경 최소

## 4. 플로우 변경 사항

```
[상황 6개 선택] → [주제 9개 선택] → [명언 맛보기 3개]
     1/3              2/3              3/3
```

### After (제안)
```
[상황 8개 선택] → [명언 맛보기 3개]
     1/2              2/2
```

### 구현 변경점

| 항목 | 변경 내용 | 난이도 |
|------|----------|--------|
| `SituationPickerScreen.tsx` | SITUATIONS 배열을 8개로 교체, 스텝 표시 "1/2"로 | 낮음 |
| `KeywordPickerScreen.tsx` | **삭제** (또는 스킵) | 낮음 |
| `QuoteTasteScreen.tsx` | keywords 파라미터 없이도 동작하도록 수정 | 낮음 |
| `preferences.ts` | keyword_groups 저장 제거 또는 상황에서 자동 매핑 | 낮음 |
| `dashboard.py` recommend API | situations만으로 추천이 충분히 작동하는지 확인 | 낮음 |
| 네비게이션 타입 | OnboardingStackParamList에서 KeywordPicker 제거 | 낮음 |

**예상 구현 시간: 2~3시간**

## 5. 향후 고도화 (v2)

온보딩 이후 **점진적 개인화**로 정밀도를 높이는 방향:

1. **행동 기반 학습** (이미 구현됨) — like/share/dwell 가중치로 프로필 자동 진화
2. **기분 체크인** (Stoic 참고) — 홈 화면에서 "오늘 기분은?" 간단 선택 → 실시간 추천 조정
3. **시간대별 자동 조정** — 아침에는 동기부여, 밤에는 성찰 명언 가중치 자동 변경
4. **온보딩 재설정** — 설정에서 "관심사 다시 선택" 기능
